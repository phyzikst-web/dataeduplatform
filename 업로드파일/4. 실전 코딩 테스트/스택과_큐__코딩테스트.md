## 문제 1. 되돌리기 마법사와 타임머신 주문

### 문제 설명
마법학과의 우수 학생인 아리는 주문서에 마법 주문을 작성하는 실습을 하고 있습니다. 하지만 아리는 덤벙거리는 성격 탓에 글자를 자주 틀리곤 합니다. 아리는 마법의 힘으로 문서 편집기의 되돌리기(Undo, Ctrl+Z)와 다시 실행(Redo, Ctrl+Y) 기능을 주문서에 적용하기로 했습니다.

아리가 사용할 수 있는 명령어는 다음과 같습니다:
1. `write X`: 주문서의 맨 뒤에 문자 `X`를 추가합니다. (X는 알파벳 소문자 한 글자)
   - 새로운 문자가 쓰여지면, 기존에 되돌리기(`undo`)를 통해 저장해 두었던 기록(Redo stack)은 모두 초기화되어 사라집니다.
2. `undo`: 주문서의 가장 마지막 글자 하나를 지우고, 이를 '되돌려진 문자 보관소'에 순서대로 저장합니다. 주문서가 비어있다면 아무 일도 일어나지 않습니다.
3. `redo`: 가장 최근에 `undo`로 지워진 글자 하나를 복구하여 주문서의 맨 뒤에 다시 적습니다. 만약 '되돌려진 문자 보관소'에 복구할 수 있는 문자가 없다면 이 명령은 무시됩니다.

모든 마법 명령을 마친 후, 최종적으로 주문서에 남은 문자들을 출력하는 프로그램을 작성하세요.

### 제한 사항
- 명령어의 개수 N은 1 이상 100,000 이하의 정수입니다. (1 <= N <= 100,000)
- write 명령어의 인자 X는 항상 알파벳 소문자 한 글자입니다.
- 주문서가 최종적으로 완전히 비어있는 경우 'EMPTY'를 출력합니다.

### 입력 형식
첫째 줄에 명령어의 개수 N이 주어집니다.
둘째 줄부터 N개의 줄에 걸쳐 각 줄에 하나의 명령어가 주어집니다.

### 출력 형식
모든 명령을 수행한 후, 최종 주문서에 남은 문자들을 공백 없이 한 줄의 문자열로 출력합니다. 주문서가 비어 있다면 'EMPTY'를 출력합니다.

### 입출력 예제

**예제 입력 1**
```
8
write a
write b
undo
write c
redo
undo
undo
write d
```

**예제 출력 1**
```
d
```

**예제 입력 2**
```
6
write x
write y
undo
redo
write z
undo
```

**예제 출력 2**
```
xy
```

### 💡 정답 및 해설

#### 모범 답안
```python
import sys

def solve():
    input = sys.stdin.read
    data = input().splitlines()
    if not data:
        return
    
    n = int(data[0])
    main_stack = []
    undo_stack = []
    
    for i in range(1, n + 1):
        command = data[i].split()
        cmd_type = command[0]
        
        if cmd_type == 'write':
            char = command[1]
            main_stack.append(char)
            undo_stack.clear()  # 새로운 글자가 쓰이면 redo 기록은 지워짐
        elif cmd_type == 'undo':
            if main_stack:
                undo_stack.append(main_stack.pop())
        elif cmd_type == 'redo':
            if undo_stack:
                main_stack.append(undo_stack.pop())
                
    if main_stack:
        print("".join(main_stack))
    else:
        print("EMPTY")

if __name__ == '__main__':
    solve()
```

#### 해설
이 문제는 후입선출(LIFO)의 특징을 가지는 스택(Stack)을 두 개 사용하여 효율적으로 해결할 수 있습니다.
1. `main_stack`: 현재 작성 중인 주문서의 글자들을 저장합니다.
2. `undo_stack`: `undo` 명령어로 인해 지워진 글자들을 역순으로 보관합니다.

각 명령어의 동작 원리는 다음과 같습니다:
- `write X`: `main_stack`에 글자를 추가(`append`)하고, 새로운 입력이 들어왔으므로 기존의 되돌리기 기록을 지우기 위해 `undo_stack.clear()`를 수행합니다.
- `undo`: `main_stack`에서 가장 최근에 들어온 글자를 꺼내어(`pop`), `undo_stack`에 쌓아둡니다(`append`).
- `redo`: `undo_stack`에 복구할 글자가 있다면 가장 최근에 취소된 글자를 꺼내어(`pop`), 다시 `main_stack`에 추가(`append`)합니다.

이 연산들은 모두 시간 복잡도 O(1)로 수행되므로 전체 N개의 명령어에 대해 O(N)의 최적의 성능으로 해결할 수 있습니다.

## 문제 2. 인쇄소의 새치기 방지 시스템

### 문제 설명
바쁜 대학교 인쇄소의 프린터는 선입선출(FIFO) 방식의 큐(Queue)를 사용하여 문서를 출력합니다. 하지만 인쇄소 사장님은 중요도가 높은 문서를 먼저 인쇄하기 위해 다음과 같은 특별한 '우선순위 대기열 규칙'을 도입했습니다.

1. 인쇄 대기열의 가장 앞에 있는 문서(Front)를 꺼냅니다.
2. 꺼낸 문서보다 중요도가 높은 문서가 대기열 뒤쪽에 하나라도 남아있다면, 이 문서를 인쇄하지 않고 대기열의 가장 뒤(Rear)로 다시 보냅니다.
3. 만약 대기열 내에 현재 문서보다 중요도가 높은 문서가 없다면, 이 문서를 즉시 인쇄합니다.

예를 들어 대기열에 4개의 문서 [A, B, C, D]가 있고 중요도가 각각 [1, 2, 3, 4]라면, D(4)가 가장 먼저 인쇄되고 그 다음 C(3), B(2), A(1) 순으로 인쇄됩니다.

여러분은 특정 문서가 주어졌을 때, 이 문서가 사장님의 새로운 규칙에 따라 몇 번째로 인쇄되는지 구하는 프로그램을 작성해야 합니다.

### 제한 사항
- 테스트 케이스의 수 T가 주어집니다.
- 각 테스트 케이스에서 문서의 개수 N은 1 이상 100 이하의 정수입니다. (1 <= N <= 100)
- 궁금한 문서의 초기 위치 M은 0 이상 N-1 이하의 정수입니다. (0 <= M < N, 0은 가장 처음 대기열의 맨 앞을 의미합니다.)
- 각 문서의 중요도는 1 이상 9 이하의 정수이며, 숫자가 클수록 중요도가 높음을 의미합니다.

### 입력 형식
첫째 줄에 테스트 케이스의 수 T가 주어집니다.
각 테스트 케이스는 두 줄로 이루어져 있습니다.
테스트 케이스의 첫째 줄에는 문서의 개수 N과 인쇄 순서가 궁금한 문서의 현재 위치 M이 공백으로 구분되어 주어집니다.
둘째 줄에는 N개 문서의 중요도가 맨 앞 문서부터 순서대로 공백으로 구분되어 주어집니다.

### 출력 형식
각 테스트 케이스에 대해, 지정된 문서가 몇 번째로 인쇄되는지 한 줄에 하나씩 출력합니다.

### 입출력 예제

**예제 입력 1**
```
3
1 0
5
4 2
1 2 3 4
6 0
1 1 9 1 1 1
```

**예제 출력 1**
```
1
2
5
```

---

### 💡 정답 및 해설

#### 모범 답안
```python
import sys
from collections import deque

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    
    num_test_cases = int(data[0])
    idx = 1
    results = []
    
    for _ in range(num_test_cases):
        n = int(data[idx])
        m = int(data[idx+1])
        idx += 2
        
        priorities = [int(x) for x in data[idx:idx+n]]
        idx += n
        
        # (중요도, 초기 인덱스) 형태로 큐에 저장
        queue = deque([(p, i) for i, p in enumerate(priorities)])
        print_order = 0
        
        while queue:
            current = queue.popleft()
            # 현재 문서보다 중요도가 높은 문서가 대기열에 있는지 확인
            if any(current[0] < item[0] for item in queue):
                queue.append(current)
            else:
                print_order += 1
                if current[1] == m:
                    results.append(str(print_order))
                    break
                    
    print("\n".join(results))

if __name__ == '__main__':
    solve()
```

#### 해설
이 문제는 선입선출(FIFO)의 구조를 가진 큐(Queue)를 시뮬레이션하여 해결하는 문제입니다. 파이썬의 `collections.deque`를 사용하면 성능 저하 없이 앞단에서의 데이터 추출(`popleft`)과 뒷단으로의 삽입(`append`)을 O(1)에 수행할 수 있습니다.

각 문서의 중요도와 함께 처음 대기열에서의 인덱스 정보를 튜플 `(중요도, 초기 인덱스)` 형태로 묶어 덱(Deque)에 보관합니다.
1. 큐의 가장 앞 원소를 꺼냅니다.
2. 큐에 남아있는 문서들 중 현재 문서보다 중요도가 높은 문서가 있는지 검사합니다.
3. 존재한다면 현재 문서를 다시 큐의 맨 뒤로 보냅니다.
4. 존재하지 않는다면 인쇄를 확정하고 인쇄 순서(`print_order`)를 1 증가시킵니다. 이때 인쇄된 문서의 초기 인덱스가 우리가 찾고자 했던 위치 `M`과 일치한다면 시뮬레이션을 중단하고 결과를 기록합니다.

## 문제 3. 회문 수집가의 덱 (Deque) 게임

### 문제 설명
회문(Palindrome)이란 '토마토'나 '기러기'처럼 앞으로 읽으나 뒤로 읽으나 똑같은 단어를 말합니다. 문자열 수집가 민우는 알파벳 소문자로 이루어진 문자열 카드를 모으는 취미가 있습니다.

민우는 양방향에서 삽입과 삭제가 모두 자유로운 자료구조인 덱(Deque)을 활용하여 문자열이 회문인지 판별하는 게임을 하려고 합니다. 하지만 어떤 단어들은 아쉽게 한 글자 차이로 회문이 되지 못합니다. 민우는 이 점이 아쉬워 딱 한 글자를 지워서 회문으로 만들 수 있는 단어를 '유사 회문(Pseudo-palindrome)'이라고 부르기로 했습니다.

민우가 가진 문자열이 주어졌을 때, 덱의 성질을 이용하여 다음 세 가지 중 하나로 분류하는 프로그램을 작성하세요:
- 주어진 문자열이 그 자체로 회문이면 `0`을 출력합니다.
- 문자열에서 단 한 글자를 삭제하여 회문을 만들 수 있는 유사 회문이면 `1`을 출력합니다.
- 두 경우 모두 해당하지 않는 일반 문자열이면 `2`을 출력합니다.

### 제한 사항
- 테스트 케이스의 수 T가 주어집니다. (1 <= T <= 30)
- 각 줄에 주어지는 문자열의 길이는 3 이상 100,000 이하입니다.
- 문자열은 알파벳 소문자로만 구성되어 있습니다.

### 입력 형식
첫째 줄에 테스트 케이스의 개수 T가 주어집니다.
둘째 줄부터 T개의 줄에 걸쳐 판별하고자 하는 문자열이 한 줄에 하나씩 주어집니다.

### 출력 형식
각 테스트 케이스에 대해 회문이면 `0`, 유사 회문이면 `1`, 둘 다 아니면 `2`를 한 줄에 하나씩 출력합니다.

### 입출력 예제

**예제 입력 1**
```
3
abba
summuus
xxyy
```

**예제 출력 1**
```
0
1
2
```

### 💡 정답 및 해설

#### 모범 답안
```python
import sys

def is_palindrome_range(s, left, right):
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True

def check_palindrome(s):
    left = 0
    right = len(s) - 1
    
    while left < right:
        if s[left] == s[right]:
            left += 1
            right -= 1
        else:
            # 불일치가 발생했을 때, 왼쪽 문자를 지우거나 오른쪽 문자를 지웠을 때 회문이 되는지 확인
            # 이는 덱의 양쪽 끝에서 비교하다 불일치 시 한쪽을 건너뛰는 흐름과 동일합니다.
            if is_palindrome_range(s, left + 1, right) or is_palindrome_range(s, left, right - 1):
                return 1
            else:
                return 2
    return 0

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    
    t = int(data[0])
    results = []
    for i in range(1, t + 1):
        results.append(str(check_palindrome(data[i])))
        
    print("\n".join(results))

if __name__ == '__main__':
    solve()
```

#### 해설
이 문제는 양쪽 끝에서 데이터를 비교하고 추출하는 덱(Deque)의 동작 방식을 투 포인터(Two Pointer) 개념으로 최적화하여 해결하는 문제입니다.

기본적인 회문 검사는 문자열의 양쪽 끝(`left`, `right`)에서 시작하여 문자가 일치하는지 확인하며 좁혀나갑니다. 

만약 `s[left] != s[right]`인 불일치 상황이 발생하면, 다음 두 가지 경우의 수를 검사합니다:
1. 왼쪽 문자 `s[left]`를 제거하고 남은 부분(`s[left+1] ~ s[right]`)이 회문인지 확인합니다.
2. 오른쪽 문자 `s[right]`를 제거하고 남은 부분(`s[left] ~ s[right-1]`)이 회문인지 확인합니다.

두 경우 중 하나라도 회문이 된다면 한 글자만 지워서 회문을 만들 수 있는 '유사 회문'이므로 `1`을 반환합니다. 둘 다 회문이 되지 못한다면 `2`를 반환합니다. 끝까지 불일치가 발생하지 않았다면 완전한 회문이므로 `0`을 반환합니다.

이 방식은 문자열의 길이 L에 대하여 최대 한 번의 분기 검사만 수행하므로 O(L)의 매우 빠른 성능으로 모든 문자열을 판별해 낼 수 있습니다.
